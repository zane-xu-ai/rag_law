## 2. 如何理解「前后」

- **B → C** 固定 **目标滑窗重叠** `CHUNK_OVERLAP=100`，仅把块链重叠**下界**从 **100** 改为 **60**（即你描述的「允许重叠掉到 60」）。
  - **首不完整**：8 → **3**（减少 5 块；主要因不再为凑满 100 重叠而 **clamp 左移起点**）。
  - **clamp 次数**：6 → **0**（句界对齐后的自然重叠已全部 ≥ 60，无需再左移）。
  - **尾不完整**：1（与重叠下界关系不大，多为逗号不在句界集、`adjust_end` ±30 内无句号等）。

- **A vs B/C**：你当前 `.env` 为 **`CHUNK_OVERLAP=50`**，没有「100 vs 60」的落差，因此**看不到**「下界 60」带来的收益；A 的统计与此前 `宪法_边界切分完整性统计.md`（1500/50）一致。

复现命令：

```bash
# B：等价于仅设置 CHUNK_SIZE=1500、CHUNK_OVERLAP=100，不设 CHUNK_OVERLAP_MIN
PYTHONPATH=src uv run python scripts/analyze_boundary_chunk_integrity.py data/宪法.md \
  --chunk-size 1500 --overlap 100 --max-probe 30 \
  -o doc/chunk/宪法_边界切分_CHUNK1500_O100_floor100.md

# C：等价于再加 CHUNK_OVERLAP_MIN=60
PYTHONPATH=src uv run python scripts/analyze_boundary_chunk_integrity.py data/宪法.md \
  --chunk-size 1500 --overlap 100 --overlap-floor 60 --max-probe 30 \
  -o doc/chunk/宪法_边界切分_CHUNK1500_O100_floor60.md
```

## 4. 结论（简要）

在 **`CHUNK_OVERLAP=100` 且 `CHUNK_OVERLAP_MIN=60`** 时，相对 **下界=100**，《宪法》上 **首不完整块数明显下降**，且 **clamp 归零**，说明「重叠可浮动到下界 60」的改动在**该参数组合**下会直接影响切分结果；**尾不完整**仍可能由句界规则与 `max_probe` 限制单独产生，需另策（如扩大 `max_probe`、扩展句界字符等）。
