## 2. 离线指标（同语料、可脚本化）

与 v1.1.7 §2、§3 及既有 chunk 报告风格对齐，建议至少记录：

| 指标 | 含义 | 备注 |
| --- | --- | --- |
| `n_chunks` | 切分后块数 | 同 `max_chars` 等后处理需对齐或单独说明。 |
| `len_min` / `p50` / `p90` / `max` | 块长分布（字符） | 发现过度切碎或超长块。 |
| `merge_ratio`（仅 B） | 语义合并率 | 来自现有合并逻辑日志或前后块数比。 |
| `breakpoint_edges`（仅 C） | 相邻窗余弦低于阈值的边数 | 见 `breakpoint_embed_diagnostics_from_pack` / 笔记本诊断。 |
| `clause_hits`（可选） | 预设锚点串（如「第[一二三四五六七八九十]+条」）在块内的命中与跨块断裂数 | 人工或正则抽样；用于法规「条」完整性。 |
| `reconstruct_ok` | 去分隔条拼接是否等于原文 | 方案 C 在 `split_overlap=0` 时应为真；D 依赖实现是否引入重叠。 |

## 3. 离线流程（建议顺序）

1. **锁定环境**：`.env` 中 `BGE_M3_PATH`、`DOCUMENT_SEGMENTATION_PATH`（仅 D）与一次 `uv sync` 版本写入报告附录。  
2. **跑 B**：对子集走与生产一致的 `iter_text_slices` + 合并；导出或打印块列表（可沿用 [doc/chunk/](../chunk/) 统计方式）。  
3. **跑 C**：对同一文件运行 `c03_breakpoint_export` 或笔记本 [`notebooks/c09_breakpoint_chunk.ipynb`](../../notebooks/c09_breakpoint_chunk.ipynb)，保存导出 md 与参数快照。  
4. **跑 D**：按 v1.1.9 流水线产出块；再施加与 C **相同的** `min_chars`/`max_chars`（若设计如此），否则在报告中明确差异。  
5. **填对比表**，对差异最大的 2～3 个文件做**人工读块**（是否切断在「条」「款」中间）。

## 4. 在线 / 抽样评测（检索质量）

目标：验证「离线块边界更好」是否转化为「召回片段更可答」。

| 步骤 | 说明 |
| --- | --- |
| 固定索引别名 | 为 B/C/D 各建独立 ES 索引或 `scope`/版本字段，避免互相覆盖。 |
| 固定 query 集 | 10～30 条：一半来自法规条文直问，一半场景问（需跨条综合）。 |
| 记录 | 每条 query：top-k 标题/片段、是否覆盖正确「条」、答案是否需拼接多块。 |
| 结论 | 以表格汇总「命中可接受 / 需调切块」比例；不单独以 nDCG 替代人工判断，除非已有标注。 |

---

## 5. 与「模型选型」的关系

- v1.1.7 **不指定**某一 HuggingFace/ModelScope SKU；若使用通用中文分段模型（如 `nlp_bert_document-segmentation_chinese-base`），在本清单下与 C 对比仍可能**不输或更差**，以本文件 §2～§4 结果为准。  
- 若法规域明显劣于 C，再考虑领域微调或放弃 D，而不是调大 `max_chars` 掩盖边界问题。
