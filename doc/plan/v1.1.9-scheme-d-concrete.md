## 1. 目标与边界

**目标**：在单篇法规 md 正文中，用 `DOCUMENT_SEGMENTATION_PATH` 指向的 **中文文档分段** 模型预测 **语义边界**（句间/子句间是否断开），再与 **最小块长 / 最大块长 / 可选重叠** 组合为 `TextChunk` 列表，并支持 **离线导出 md**（与方案 C 的 `c03_breakpoint` 对称，便于 diff）。

**明确不做（本版本）**

- 不把分段向量与 BGE 对齐；检索仍只用 BGE。
- 不默认替代方案 B/C；仅作为可选切块策略或实验分支。
- 不在未跑完 v1.1.8 评测前接入 `rag_ingest` 主链路。

## 2. 模型与路径（推荐缺省）

| 项 | 建议 |
| --- | --- |
| 权重 | 本地目录或 ModelScope `iic/nlp_bert_document-segmentation_chinese-base` 等价物；与 `.env` 中 `DOCUMENT_SEGMENTATION_PATH` 一致。 |
| 任务接口 | `document-segmentation` 类 pipeline：对**按句或按行切分**的单元序列输出 **是否 boundary**（以模型卡为准：多为相邻单元间二分类或 BIO）。 |
| 推理栈 | 优先 **transformers + PyTorch**（与现有 Python 栈一致）；可选 ONNX 降延迟（后续）。 |
| 资源 | 与 BGE **分进程或分阶段加载**，避免显存峰值叠加；记录 `infer_ms_per_1k_chars` 供入库 SLA 评估。 |

## 3. 数据流（与方案 C 后处理对齐）

```mermaid
flowchart LR
  md[full_md_text]
  units[细粒度单元序列]
  model[doc_segmentation_model]
  raw_ranges[模型边界区间]
  merge_min[_merge_ranges_min_len]
  split_max[_split_range_max_len]
  chunks[TextChunk列表]

  md --> units
  units --> model
  model --> raw_ranges
  raw_ranges --> merge_min
  merge_min --> split_max
  split_max --> chunks
```

1. **输入**：全文字符串（与 MinerU md 一致；不切头去尾除非与 B/C 统一约定）。  
2. **单元**：将正文拆成模型要求的粒度（常见为 **按句** `\n` 或 `。` 规则 + 长度上限；**不得**与 C 的滑动字窗混用）。  
3. **模型**：对相邻单元对或序列打标签，得到 **字符级边界** 列表 `boundaries`（与 `starts` 对齐方式需在实现中固定并写单测）。  
4. **原始区间**：相邻边界之间形成 `[s, e)`，与 [`raw_breakpoint_ranges`](../../src/chunking/breakpoint_embed.py) 地位类似（语义段、未合并）。  
5. **后处理**：**复用** [`_merge_ranges_min_len`](../../src/chunking/breakpoint_embed.py) 与 [`_split_range_max_len`](../../src/chunking/breakpoint_embed.py)（或抽成 `chunking/length_constraints.py` 公共函数），参数与方案 C 对齐：`min_chars`、`max_chars`、`split_overlap`，便于与 `宪法_c03_breakpoint_golden.json` **公平对比**（仅边界来源不同）。  
6. **输出**：`TextChunk`（`chunk_index`、`char_start`/`char_end`、`source_path` 等）；可选 `extra={"chunking": "document_segmentation_d", ...}`。

## 4. 配置与命名（建议）

| 变量 / Settings 字段 | 含义 |
| --- | --- |
| `DOCUMENT_SEGMENTATION_PATH` | 已有；模型根目录。 |
| `CHUNK_DOC_SEGMENTATION_ENABLED`（可选） | 默认 `false`；为 `true` 且路径有效时走 D（仅当未来接入 ingest）。 |
| `DOC_SEGMENTATION_MIN_CHARS` / `MAX_CHARS` / `SPLIT_OVERLAP` | 与 C 的 `MIN_CHARS`/`MAX_CHARS`/`SPLIT_OVERLAP` **默认值一致**，便于对比；可在 `.env` 单列前缀避免混淆。 |

导出目录建议：`data/chunk_md/d04_document_segmentation/`，脚本名建议：`scripts/d04_document_segmentation_export.py`（与 `c03_breakpoint_export` 对称）。

## 7. 风险与回滚

| 风险 | 缓解 |
| --- | --- |
| 通用模型在法规上边界过碎/过粗 | 阈值与后处理与 C 对齐后再判；考虑领域微调（排期外）。 |
| 显存 / 延迟 | 与 BGE 分时加载；批大小与单元粒度可配。 |
| 与检索空间不一致 | 接受 v1.1.7 设计；以召回与答案质量为准。 |

回滚：关闭开关或停用 `d04` 导出；主链路未接时零影响。
