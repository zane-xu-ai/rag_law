# v1.1.2 压测：Locust 对 QA WebUI（SSE）负载与容量

| --- | --- |

## 1. 目标与范围

- **目标**：在可重复、可配置的前提下，测量 QA 问答服务在不同并发与请求速率下的 **成功率、端到端耗时、流式响应完整性**，并结合已有监控（`logs/monitor.log`、ES 监控索引、Grafana）做关联分析。
- **范围**：选用 **Locust** 编写压测场景、执行步骤、风险与成本控制；附常见压测工具对比供选型参考。
- **不在本版必选项**：对 Elasticsearch 集群或 LLM 供应商做极限压测（易产生费用与封限）；生产环境未经审批的全量压测。

---

## 2. 压测工具简单对比

以下从「本仓库技术栈（Python）、SSE 场景、学习成本」角度做简要对比，**无绝对优劣**，按团队习惯与目标选型即可。

| 工具 | 实现/语言 | 典型优点 | 典型缺点 | 与 SSE / 流式 |
| --- | --- | --- | --- | --- |
| **Locust** | Python + gevent | Web UI **运行中可调**用户数与 spawn 速率；脚本即 Python，易与 `requests` 结合读流；与仓库依赖同源 | 单机极限 RPS 通常低于 k6；分布式需多 worker 与带宽规划 | **适合**：`stream=True` 逐块读至结束或按字节上限截断 |
| **k6** | Go，脚本类 JS | 单机 **高并发**、资源占用低；内置 `stages`、阈值、报告成熟 | 脚本非 Python；对流式响应需自行处理 body 读完或超时 | 可用，但流式逻辑不如 Python 直观 |
| **wrk / hey** | C / Go | 极轻、命令行一行起压 | 场景与断言弱，不适合复杂 body、SSE 细粒度统计 | 一般只做「整段响应时间」，对流式语义弱 |
| **Apache JMeter** | Java / GUI | 生态大、企业报表与插件多 | 重、维护成本高；与本仓库 Python 栈不一致 | 有 HTTP 采样器，SSE 需插件或 BeanShell |

**本计划选用 Locust 的理由（摘要）**：

1. 与项目同为 **Python**，压测脚本纳入 `dev` 依赖，与主工程一致。
2. **Web UI**（默认 **8089**）可在压测过程中调整并发。
3. SSE 通过 `HttpUser` 客户端 **`stream=True` + `iter_content`** 读满流后计一次响应时间，与「整次问答结束」语义一致。

## 3. 压测对象与接口

| 项 | 说明 |
| --- | --- |
| 服务 | `qa.webui.app:app`（默认端口 **8766**，脚本 [`scripts/start_qa_webui.sh`](../../scripts/start_qa_webui.sh)） |
| 接口 | `POST /api/qa/stream` |
| Content-Type | `application/json` |
| Body 示例 | `{"query":"测试问题","k":5,"max_tokens":128}`（压测时宜**降低 `max_tokens`**，控制费用与单次时长） |
| 响应 | `text/event-stream`（SSE），脚本侧读满流为止或读超时 |

**健康检查（可选）**：`GET /api/health`，可用于压测前探活。

---

## 4. 前置条件与风险

1. **LLM 费用**：每轮请求会调用真实（或兼容）API，压测前必须估算 `并发 × 轮数 × token` 上限；使用**低 `max_tokens`**、**短 query**、**限总时长**。
2. **本机资源**：BGE 与 LLM 同进程时，CPU/GPU/内存易成为瓶颈；结论应注明环境（机型、是否 GPU）。
3. **ES**：检索负载随并发上升；避免对生产业务索引做未授权高压。
4. **合规**：query 勿含真实用户隐私；使用测试账号与 API 限额。

## 5. 落地说明（已实现）

### 5.1 依赖

- **`pyproject.toml`**：`locust>=2.32,<3` 已加入 **`[dependency-groups].dev`** 与 **`[project.optional-dependencies].dev`**（与 `pytest` 等并列，默认 `uv sync` 即安装）。
- 安装：`uv sync`（或 `pip install -e ".[dev]"`）。

### 5.2 交付文件

| 路径 | 说明 |
| --- | --- |
| [`scripts/load/locust/locustfile.py`](../../scripts/load/locust/locustfile.py) | `QAStreamUser`：`POST /api/qa/stream`，`stream=True`，`iter_content` 读完全部 SSE 后计成功；报表中响应时间为**整流耗时**。 |
| [`scripts/load/locust/README.md`](../../scripts/load/locust/README.md) | 命令、环境变量、风险说明（与本文 **§6** 一致，可任选其一查阅）。 |

### 5.3 实现要点

- **基类**：`HttpUser`，未使用 `FastHttpUser`（需完整读 SSE 流，`requests` 后端行为清晰）。
- **负载形状**：`StagesShape`（`LoadTestShape`）仅在环境变量 **`LOCUST_USE_SHAPE=1`** 时注册；默认不启用，便于用 **Web UI 手动设并发**。启用阶梯时，默认三阶段为 60s@2 → 120s@5 → 180s@10 用户，可通过 `LOCUST_SHAPE_STAGES` 覆盖。
- **指标名**：请求统计名为 `/api/qa/stream [SSE full stream]`。

## 6. 使用方法

### 6.1 启动被测服务

需 **embedding + llm**（与主 README 一致），例如：

```bash
uv sync --extra embedding --extra llm
uv run uvicorn qa.webui.app:app --host 127.0.0.1 --port 8766
```

或使用一键脚本（含可选 Filebeat）：[`scripts/start_qa_webui.sh`](../../scripts/start_qa_webui.sh)（默认端口 **8766**、监听 `0.0.0.0` 时，压测机访问请改为本机 IP 或 `127.0.0.1`）。

### 6.2 探活与手工 SSE 校验

```bash
curl -sSf http://127.0.0.1:8766/api/health
curl -N -H "Content-Type: application/json" -d '{"query":"hi","max_tokens":64}' \
  http://127.0.0.1:8766/api/qa/stream
```

### 6.3 启动 Locust（Web UI 调并发）

```bash
uv run locust -f scripts/load/locust/locustfile.py --host http://127.0.0.1:8766
```

浏览器打开 **<http://localhost:8089>**，设置并发用户数与 spawn 速率。

### 6.4 阶梯升压（可选）

设置 **`LOCUST_USE_SHAPE=1`** 后，由 `StagesShape.tick` 调度用户数（Web UI 中的用户数会被忽略）：

```bash
LOCUST_USE_SHAPE=1 uv run locust -f scripts/load/locust/locustfile.py --host http://127.0.0.1:8766
```

自定义阶段（多段逗号分隔，每段 `时长秒:用户数:spawn_rate`）：

```bash
LOCUST_USE_SHAPE=1 LOCUST_SHAPE_STAGES=30:1:1,60:3:1,120:8:2 \
  uv run locust -f scripts/load/locust/locustfile.py --host http://127.0.0.1:8766
```

### 6.5 环境变量一览

| 变量 | 默认 | 说明 |
| --- | --- | --- |
| `LOCUST_QA_QUERY` | 短中文问句 | 请求体 `query` |
| `LOCUST_QA_K` | `5` | 检索条数；`0` 表示不传 `k` |
| `LOCUST_QA_MAX_TOKENS` | `128` | 压测宜压低，控制费用与时长 |
| `LOCUST_CONNECT_TIMEOUT_S` | `10` | 连接超时（秒） |
| `LOCUST_READ_TIMEOUT_S` | `600` | 读 SSE 总超时（秒） |
| `LOCUST_USE_SHAPE` | 未设置 | 设为 `1` 启用阶梯升压 |
| `LOCUST_SHAPE_STAGES` | 见 `locustfile.py` | 阶梯形状，见 §6.4 |

示例：

```bash
LOCUST_QA_MAX_TOKENS=64 LOCUST_QA_QUERY="你好" \
  uv run locust -f scripts/load/locust/locustfile.py --host http://127.0.0.1:8766
```

### 6.6 无界面快速跑（可选）

```bash
uv run locust -f scripts/load/locust/locustfile.py --headless -u 1 -r 1 -t 30s \
  --host http://127.0.0.1:8766
```
