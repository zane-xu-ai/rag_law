## 1. 目标

在现有 **配置层**（[`src/conf/settings.py`](../../src/conf/settings.py) 已含 `BGE_M3_PATH`、`EMBEDDING_BATCH_SIZE`）与 **切分**（[`src/chunking/`](../../src/chunking/)）之上，新增 **`src/embeddings`**：

- 从本地路径加载 **BGE-M3**（`FlagEmbedding` 的 `BGEM3FlagModel` 或经评估的等价方式）。
- 提供 LangChain 风格的 **`embed_documents` / `embed_query`**，供后续入库与在线检索共用。
- 向量数值与 Elasticsearch **`dense_vector` + `similarity: cosine`** 约定一致：**对 dense 输出做 L2 归一化**后再供写入与 kNN（详见 [v1.0.0 计划 §6](v1.0.0-rag-law-mvp-plan.md)）。

**不在本版本**：Qwen3 Embedding 等其它后端、稀疏向量写入 ES、混合检索编排。

---

## 2. 行为契约

| 项 | 约定 |
| --- | --- |
| API | `embed_documents(texts: list[str]) -> list[list[float]]`（与 `texts` 等长）；`embed_query(text: str) -> list[float]` |
| 模型路径 | `BGE_M3_PATH`（与 Settings 一致） |
| 批大小 | `EMBEDDING_BATCH_SIZE` 切分 `embed_documents` |
| query / 文档 | **分路编码**：共用同一 `BGEM3FlagModel` 实例，`embed_query` 与 `embed_documents` 分别按 [FlagEmbedding](https://github.com/FlagOpen/FlagEmbedding) 对检索场景推荐的参数调用 `encode`（具体参数以所用版本文档为准） |
| 归一化 | 应用层 **显式 L2 归一化**，作为单一事实来源（即使库内已归一化） |
| 维度 | 只读 **`dense_dimension`**：在首次成功编码后确定（常见 **1024**），供后续 `indices.create` 的 `dims` 使用，避免硬编码与真实模型不一致 |

## 3. 代码与依赖

### 3.1 目录与职责

| 路径 | 职责 |
| --- | --- |
| [`src/embeddings/base.py`](../../src/embeddings/base.py) | `EmbeddingBackend`（`Protocol` 或 ABC）：`embed_documents`、`embed_query`、`dense_dimension` |
| [`src/embeddings/bge_m3.py`](../../src/embeddings/bge_m3.py) | `BgeM3EmbeddingBackend`：加载模型、批处理、L2、query/doc 分路 |
| [`src/embeddings/__init__.py`](../../src/embeddings/__init__.py) | 导出类型；`build_embedder(settings) -> EmbeddingBackend`（当前仅返回 BGE 实现） |

可选：预留 `EmbeddingVector`（`dense` 必填、`sparse` 可选）数据类，MVP 可仅用 dense 路径（与 [BGE-M3 向量层实现计划 §3](../text2vector/BGE-M3%20向量层实现计划.md) 扩展性说明一致）。

### 3.2 依赖

- 在 [`pyproject.toml`](../../pyproject.toml) 增加 **`embedding`** optional-dependencies（例如 `FlagEmbedding`、`torch`、`numpy`），**不**加入默认 `dependencies`，保证未装向量环境时仍可跑 `conf` / `chunking`。
- `bge_m3` 中对 `torch` / `FlagEmbedding` 的导入应使「未安装 extra」时 `import embeddings` 仍可选（或仅在 `build_embedder` / 首次构造时失败并提示 `pip install -e ".[embedding]"`）。

### 3.3 配置

- 复用 **`BGE_M3_PATH`**、**`EMBEDDING_BATCH_SIZE`**。
- 可选：`VECTOR_SIMILARITY=cosine`（默认）写入 [`.env.example`](../../.env.example)，与后续 ES 模块对齐；本阶段可不读该变量，仅在文档中约定。

## 4. 测试与覆盖率

- **单元测试**（[`tests/test_embeddings/`](../../tests/test_embeddings/)）：mock `BGEM3FlagModel` 或 fake backend，断言形状、批行为、L2 范数约为 1。
- **可选集成测试**：`pytest.mark.integration`，环境变量与模型路径存在时对小样本真实编码。

在 [`pyproject.toml`](../../pyproject.toml) `[tool.coverage.report] include` 中纳入 `*/embeddings/*`（与 `conf` 等并列）。

## 5. 文档与验收

- [`.env.example`](../../.env.example) / [README.md](../../README.md)：说明向量功能需安装 `embedding` extra（如 `uv sync --extra embedding`）。
- 实施完成后：更新 [doc/text2vector/BGE-M3 向量层实现计划.md](../text2vector/BGE-M3%20向量层实现计划.md) §7 任务勾选与状态；在 [v1.0.0-rag-law-mvp-plan.md](v1.0.0-rag-law-mvp-plan.md) §9 将「向量」项勾选，并注明归一化与 ES `cosine` 一致。

**验收**：在已安装 `embedding` extra 且 `BGE_M3_PATH` 指向有效模型时，`build_embedder(get_settings())` 后 `embed_query("测试")` 返回长度为 `dense_dimension` 的 float 列表，且 L2 范数约为 1。
