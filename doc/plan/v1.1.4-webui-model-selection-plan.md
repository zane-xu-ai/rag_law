# v1.1.4 WebUI 模型选择：供应商/模型二级选择与默认回退

| 属性 | 说明 |
| --- | --- |
| 文档版本 | v1.1.4 |
| 状态 | 已实施 |
| 前置 | v1.0.7 QA WebUI、v1.1.1 监控、v1.1.3 Token 成本观测 |
| 适用范围 | `src/qa/webui` 前端选择模型、后端模型白名单校验、异常可视化与默认模型兜底 |

---

## 1. 目标

在 QA WebUI 中提供可用模型的可视化选择能力，并保证在异常场景下服务可用：

- 支持**二级选择**：一级供应商，二级模型名。
- 模型列表来自配置文件，不走静态文件直链。
- 当模型列表不可用或用户选择非法模型时，自动回退到 `.env` 的 `MODEL_NAME`。
- 后端任意阶段异常（LLM/向量化/检索）要及时在前端展示，避免“页面一直卡住”。

---

## 2. 配置与数据源设计

### 2.1 配置项

在 `Settings` 中新增：

- `MODEL_CONFIG_PATH`（默认：`settings/qwen_model/qwen_free_model_name_ranked.json`）

并新增能力：

- `model_config_path_resolved`：解析绝对路径
- `load_model_config()`：启动时读取 JSON 到内存（失败返回空字典）

### 2.2 模型数据结构

模型配置按供应商分组，核心字段为：

- `rankedModels.<provider> = [modelA, modelB, ...]`

前端按该结构渲染二级下拉。

---

## 3. 后端实现（已落地）

### 3.1 启动时加载到内存

`src/qa/webui/app.py` 在 `lifespan` 中加载并缓存：

- `app.state.model_config = settings.load_model_config()`

### 3.2 模型配置 API

- `GET /api/models`
  - 返回内存中的模型配置
  - 若未加载成功，返回 404

### 3.3 模型白名单与默认回退

`POST /api/qa/stream` 行为：

- 若请求体携带 `model`：
  - 仅当该模型存在于 `model_config.rankedModels` 时允许透传；
  - 否则记录 warning 并回退为默认模型（即不传 `model_override`，最终使用 `settings.model_name`）。

### 3.4 异常可视化（防卡死）

SSE 生成器最外层增加统一异常兜底：

- 捕获 `stream_qa_events` 的未处理异常；
- 向前端发送：
  - `type=error`（含错误类型与信息）
  - `type=done, ok=false`

保证前端能结束“处理中”状态，按钮恢复可用。

---

## 4. 前端实现（已落地）

文件：`src/qa/webui/static/index.html`

- 新增供应商与模型下拉：
  - `#provider`
  - `#model`
- 页面初始化调用 `/api/models`，填充二级菜单。
- 用户提交时：
  - 若选择了模型，则请求体携带 `model`；
  - 若未选择或加载失败，不携带 `model`，走后端默认模型。
- 模型列表加载失败时：
  - 下拉置灰并显示“默认模型”提示；
  - 不阻断“提交提问”。

---

## 5. 验收标准

- 页面可看到供应商列表，切换供应商后模型列表联动更新。
- 选择模型提交后，后端按该模型调用（且受白名单校验）。
- 模型配置不可用时，仍可正常提问并走默认模型。
- 出现 LLM 403、向量化错误、检索错误时：
  - 前端错误条可见具体错误；
  - 页面不会卡死，“提交提问”可恢复点击。

## 7. 运维注意事项

- 若修改模型配置文件，需重启 WebUI 使内存缓存生效。
- `MODEL_CONFIG_PATH` 可切换到不同模型清单文件，便于分环境发布。
- 前端只负责“可选项体验”，最终安全控制应以后端白名单为准。
