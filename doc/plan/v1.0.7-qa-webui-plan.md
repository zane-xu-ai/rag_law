## 1. 目标

1. 在 [`src/qa/webui`](../../src/qa/webui) 提供 Web 界面：展示用户问题、RAG 最终答案（流式），以及**可理解的中间过程**（如：向量化、ES 检索、命中条数、大模型生成中）。
2. **耗时可观测性（硬性要求）**  
   - **步骤尽量拆细**：凡可独立计时的子步骤，均应**单独输出该步耗时**（毫秒级，建议 `elapsed_ms` 或等价字段），避免只给一个「总 RAG 耗时」或「总 LLM 耗时」而无法定位瓶颈。  
   - **首 token 耗时（TTFT）**：从「本请求在服务端被接受并开始处理」到「大模型流式响应中收到第一个有效 token（或第一个含内容的 delta）」之间的时长；须在结果或事件流中**显式给出**（字段名实现时自定，如 `ttft_ms` / `first_token_ms`）。  
   - **总耗时**：从同一起点到「本请求全流程结束」（流结束、连接关闭或收到 `done` 类终态事件）的时长；须**显式给出**（如 `total_ms`）。  
3. **单轮优先**：当前产品只考虑**一轮**用户输入与一轮助手回复；不在首版实现会话记忆、历史列表 UI 或多轮上下文拼装。  
4. **多轮扩展预留**：协议或 API 形状上预留 `conversation_id`、多轮 `messages` 等扩展可能，首版可为空或忽略，避免日后推翻接口。

---

## 2. 非目标（首版）

- 多轮对话的完整产品与实现（仅预留扩展点，见第 7 节）。  
- 与「思考过程」相关的展示：仅当所选 LLM 接口在流式协议中**明确区分**推理与最终答案时再行对接；本计划不强制、不规定具体字段名（避免与未选定的供应商绑定）。  
- 鉴权、多租户、公网部署安全加固（可作为后续专项）。

---

## 3. 架构原则（仅概念）

- **浏览器 → 服务端**：建议一次提交用户问题（单轮），服务端通过**长连接推送**（如 SSE）向下游顺序发送「阶段事件」「检索摘要」「流式 token」「终态与耗时汇总」。  
- **计时起点**：建议统一为**服务端开始处理该次问答请求的时刻**（例如进入 RAG 流水线入口的单调时钟时刻），所有相对耗时均相对该点或可分解为相邻子步骤之和，并在文档/实现说明中写清，避免与「用户点击时刻」混用。  
- **首 token 终点**：以流式 API 返回的**第一个非空内容增量**为准；若仅有空 chunk，以实现约定是否跳过直至首个有效字符。

---

## 4. 步骤分解与「尽量细」的耗时要求

下列为建议的**最小观测粒度**；实施时**不得**随意合并到更粗粒度，除非技术上无法分离（须在实现说明中注明）。每一步均输出**该步独立 `elapsed_ms`**（从该步开始到该步结束，或采用「阶段开始/结束」成对事件，由前端差分亦可，但服务端应能给出明确数值）。

| 序号 | 阶段（建议 id） | 说明 | 是否必须单独耗时 |
| --- | --- | --- | --- |
| S0 | `request_accept` | 请求已进入应用层处理（可选：与流水线起点合一） | 可选 |
| S1 | `settings_resolve` | 解析/加载本次请求所需配置（若与请求绑定） | 建议 |
| S2 | `embedder_acquire` | 获取编码器实例（若存在懒加载、首次建连） | 建议 |
| S3 | `embed_query` | 仅查询向量 `embed_query(query)` | **必须** |
| S4 | `es_client_acquire` | 获取 ES 客户端（若与 S3 可完全并行则注明） | 建议 |
| S5 | `es_search_knn` | `search_knn` 往返（不含 S4 若已单列） | **必须** |
| S6 | `build_messages` | 拼接 system/user、片段截断等 | **必须** |
| S7 | `llm_client_ready` | 构造 OpenAI 兼容客户端（若耗时不可忽略） | 可选 |
| S8 | `llm_stream_open` | 发起流式补全至**首包/首连接建立**（若与首 token 不同则拆分） | 建议 |
| S9 | `llm_first_token` | 从流水线起点到**首个有效 token**（**TTFT**） | **必须（作为指标输出）** |
| S10 | `llm_stream_body` | 首 token 之后至流结束的生成过程 | **必须**（可与 S9 合并展示为「生成阶段」，但 TTFT 与总 LLM 耗时应可由子项还原） |
| S11 | `response_finalize` | 收尾、汇总耗时、关闭流 | 建议 |

**说明**：

- 若某两步在代码中不可分割（例如 `embedder` 与 `embed_query` 严格同步且无独立边界），允许合并为一条事件，但须在计划/实现说明中**写明合并原因**。  
- **检索命中条数**、各条 `source_file` / `chunk_index` 摘要应在 `es_search_knn` 结束后的**独立事件**中输出，便于 UI 展示「检索到 N 条」；该事件可自带 `elapsed_ms`（通常含在 S5）或引用 S5 的耗时字段。

## 5. 核心耗时指标定义（须在 UI 或终态中可见）

| 指标 | 定义 | 要求 |
| --- | --- | --- |
| **TTFT（首 token 耗时）** | 自统一「流水线起点」至首个流式内容增量（有效字符）到达 | **必须输出** |
| **总耗时 `total_ms`** | 自同一「流水线起点」至请求完全结束 | **必须输出** |
| **分步耗时** | 第 4 节各可独立步骤的 `elapsed_ms` | **尽量全覆盖**；前端可用时间线或表格展示 |

可选补充（非强制）：`rag_prefill_ms`（从起点到 LLM 流开始前的累计，便于对比「检索+拼 prompt」与「纯生成」）。

---

## 6. 前端展示建议（仅产品层）

- **时间线或步骤列表**：按顺序展示 S1～S8 等阶段状态（进行中/完成）及对应耗时。  
- **检索区**：展示命中条数 N 与每条简要信息（与 [v1.0.5-qa-plan.md](v1.0.5-qa-plan.md) 中 user 结构一致即可）。  
- **答案区**：流式追加正文；**TTFT** 与 **总耗时** 在页面上醒目位置或汇总区展示（例如「首字延迟 xx ms · 全程 xx ms」）。

---

## 7. 单轮与多轮扩展预留

| 项目 | 首版（单轮） | 预留 |
| --- | --- | --- |
| 对话轮次 | 仅当前一条用户问题与一条助手回复 | 请求体或连接元数据中可预留 `conversation_id`（UUID 占位），不传则按单轮处理 |
| 上下文 | 仅本次 `query` 进入 RAG 与 messages | 预留未来 `messages: [{role, content}, ...]` 或服务端会话存储，首版可忽略或只读单条 user |
| 流式协议 | 单次 SSE/流结束即会话结束 | 终态事件可带 `conversation_id` 便于下一轮复用（实现可选） |

**不要求**首版实现会话存储、历史 UI 或上下文截断策略；仅要求**不堵死**后续多轮对接。

---

## 8. 测试与验收（规划）

- **可观测性**：用固定问题跑一次，日志或事件序列中能还原各 `elapsed_ms`、TTFT、`total_ms`，且 `total_ms` 与（可选）各阶段之和合理一致（允许实现约定内的重叠或并行说明）。  
- **单轮**：连续两次独立提问互不干扰（无必须共享状态）。  
- **性能**：不在本计划写死数值门槛；仅要求指标字段存在且语义与第 5 节一致。

## 10. 已实现摘要与运行方式

以下与当前仓库实现一致，便于对照与运维。

### 10.1 代码与职责

| 路径 | 说明 |
| --- | --- |
| [`src/qa/streaming.py`](../../src/qa/streaming.py) | `stream_qa_events(query, *, settings=None, k, max_tokens, conversation_id)`：产出供 SSE 序列化的 **JSON 事件**；`format_sse_event` 格式化为 `data: <json>\n\n`。计时起点为进入生成器后、发出 `start` 事件之前的 `time.perf_counter()` 时刻。 |
| [`src/qa/webui/app.py`](../../src/qa/webui/app.py) | FastAPI：`POST /api/qa/stream`、`GET /api/health`；根路径挂载 [`static/`](../../src/qa/webui/static/)（与 chunking WebUI 相同模式）。未安装 `openai` 时流式接口返回 **503**（提示 `uv sync --extra llm`）。 |
| [`src/qa/webui/static/index.html`](../../src/qa/webui/static/index.html) | 单页：阶段耗时列表、检索结果、流式答案，以及 **TTFT / 全程 / RAG 预填** 展示。 |
| [`src/qa/__init__.py`](../../src/qa/__init__.py) | 额外导出 `stream_qa_events`、`format_sse_event`（与 `answer_question`、`build_messages` 等并列）。 |

### 10.2 事件类型与阶段 id（`stream_qa_events`）

- **事件类型**：`start`；`phase`（`elapsed_ms`、`offset_ms`）；`retrieval`（`hit_count`、`hits` 摘要、`elapsed_ms`）；`metrics`（首个有效 token 后：`ttft_ms`、`rag_prefill_ms`）；`delta`（`content`）；`error`；`done`（`ok`、`total_ms`、`ttft_ms`、`rag_prefill_ms`、`timings`、完整 `text`、`conversation_id`）。
- **`timings` 中的阶段 id**（与上文第 4 节表对齐）：`settings_resolve`、`embedder_acquire`、`embed_query`、`es_client_acquire`、`es_search_knn`、`build_messages`、`llm_client_ready`、`llm_stream_open`、`llm_first_token_wait`、`llm_stream_body`、`response_finalize`。

### 10.3 HTTP API

- **`POST /api/qa/stream`**：请求体 JSON——`query`（必填）、`k`（可选）、`max_tokens`（可选，默认 2048）、`conversation_id`（可选，多轮预留）。响应 **`Content-Type: text/event-stream`**（SSE）。
- **`GET /api/health`**：`{"status":"ok"}`。

### 10.5 本地启动与前置条件

**依赖**（与 CLI 问答一致）：

```bash
uv sync --extra embedding --extra llm
```

**启动**（默认端口 **8766**，避免与切分预览 **8765** 冲突）：

```bash
uv run uvicorn qa.webui.app:app --host 127.0.0.1 --port 8766
```

浏览器访问：`http://127.0.0.1:8766/`。

**前置条件**：Elasticsearch 已按 [`v1.0.4-ingest-plan.md`](v1.0.4-ingest-plan.md) 入库；`.env` 中 `BGE_M3_PATH`、`MODEL_*`、`ES_*` 可用（与 [`scripts/rag_qa.py`](../../scripts/rag_qa.py) 一致）。
