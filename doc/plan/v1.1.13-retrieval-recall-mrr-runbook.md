# 检索金标评测执行说明（Recall@k / MRR）

| 属性 | 说明 |
| --- | --- |
| 版本 | v1.1.13 |
| 关联 | 指标定义见 [v1.1.11-rag-evaluation-plan.md](v1.1.11-rag-evaluation-plan.md) 第 3～4 节 |

## 1. 目的

在固定 **ES 索引快照** 与 **与线上一致的检索路径**（`embed_query` → `EsChunkStore.search_knn`）下，对金标 `queries.jsonl` 计算 **Recall@k** 与 **MRR**，并将结果落盘以便对比实验与回归。

## 2. 前置条件

- 金标文件含字段：`query`、`gold_chunk_ids`（与 ES 文档 `_id` 一致）、可选 `gold_primary_chunk_id`（MRR 主块；缺省用 `gold_chunk_ids[0]`）。
- `.env` 中 **`ES_INDEX`**、**`BGE_M3_PATH`**、**`RETRIEVAL_K`** 等与入库时一致；向量维度与索引 `dense_vector` 一致。
- 索引中已包含金标所引用的 chunk（否则指标会系统性偏低）。

## 3. 参数约定

| 参数 | 含义 |
| --- | --- |
| **k** | Recall@k 的 k；默认 `Settings.retrieval_k`（`RETRIEVAL_K`）。 |
| **mrr_k_max** | MRR 截断深度；默认 `max(20, k×2)`，与 v1.1.11 一致。 |
| **单次 ES 检索深度** | `max(k, mrr_k_max)`，一次 kNN 同时支撑两项指标。 |

## 4. 执行命令

在项目根目录：

```bash
uv run python scripts/e03_eval_retrieval_recall_mrr.py \
  --gold data/eval/gold/rag_law_d05/smoke_hunyin_async_v1/queries.jsonl
```

可选：`--k 5`、`--mrr-k-max 20`、`--limit 50`（调试）、`--output-dir ...`、`--run-id my_run`。

## 5. 输出位置

默认目录：`data/eval/results/<金标父目录名>_<UTC时间戳>/`

| 文件 | 内容 |
| --- | --- |
| `metrics.json` | `recall_at_k`、`mrr`、`k`、`mrr_k_max`、`n_queries_evaluated`、环境指纹（`es_index`、`chunk_version`、`bge_m3_path` 等） |
| `per_query.jsonl` | 逐条是否命中、MRR、主块 rank |
| `README.md` | 简要说明 |

## 6. 与报告对齐

对比不同切块或重排策略时，为每次实验保留独立子目录，并在对比表中写明：**同一 `ES_INDEX`（或注明快照/版本字段）、同一 k、mrr_k_max、同一金标路径**。
