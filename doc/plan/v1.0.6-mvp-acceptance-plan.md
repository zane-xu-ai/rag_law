| 属性 | 说明 |
| --- | --- |
| 文档版本 | v1.0.6 |
| 状态 | 已发布（对应 [v1.0.0-rag-law-mvp-plan.md](v1.0.0-rag-law-mvp-plan.md) §9「验收」） |
| 关联 | 前置：[`v1.0.4-ingest-plan.md`](v1.0.4-ingest-plan.md) 入库、`ES_INDEX` 有数据；[`v1.0.5-qa-plan.md`](v1.0.5-qa-plan.md) 问答 CLI [`scripts/rag_qa.py`](../../scripts/rag_qa.py) |
| 范围 | **人工** 用 1～2 个法律相关问题走通问答，核对 **检索命中 chunk** 与 **模型答案** 是否 **语义上贴合** 语料中的条文表述；**不含** 自动化回归、官方合规认证 |

---

## 1. 目标

在本地或联调环境确认 MVP 闭环「可解释、可核对」：

1. **检索**：top-k 命中的 `source_file`、`chunk_index` 与片段正文，应与问题主题 **相关**（能支撑回答或能诚实说明「片段不足」）。
2. **答案**：须在 system/user 约束下 **引用片段编号** 或说明无依据；结论与片段内容 **不矛盾**，**不编造** 具体条号或文义。

---

## 2. 前置条件

- Elasticsearch 已按 [`scripts/rag_ingest.py`](../../scripts/rag_ingest.py) 写入与 `.env` 中 `ES_INDEX` 一致的索引；本机可访问 `ES_HOST` / `ES_PORT`。
- 已 `uv sync --extra embedding --extra llm`，`.env` 中 `BGE_M3_PATH`、`MODEL_*` 有效。
- （可选）先跑 `uv run python scripts/llm_smoke_test.py` 确认 LLM 连通。

---

## 3. 推荐操作步骤

1. **选 1～2 个法律相关问题**（见下文 §5 示例；亦可自拟，但须能在 `data/*.md` 语料中找到可对照的表述）。
2. **先看检索上下文（推荐）**  
   `uv run python scripts/rag_qa.py "<你的问题>" --dry-run`  
   检查打印的 JSON 中 `messages[1].content`：  
   - **【检索片段】** 下各条是否 **与问题同主题**；  
   - 若命中明显无关或为空，记录为 **检索/索引问题** 或 **接受「无法律依据」类回答**（与 [v1.0.5-qa-plan.md](v1.0.5-qa-plan.md) 设计一致）。
3. **再生成答案**  
   `uv run python scripts/rag_qa.py "<你的问题>"`（可用 `--k` 调整条数）。  
   对照步骤 2 中的片段，检查答案是否：  
   - 明确引用「片段 1」「片段 2」或等价表述；  
   - 核心论断可由片段 **推出或合理概括**，无与片段 **明显冲突** 的编造。
4. **条文贴合（人工核对）**  
   在仓库 `data/<来源文件>.md` 中 **定位** 与命中块对应的章节（凭 `chunk_index` 与正文片段辅助定位），确认模型概括与原文 **无实质性歪曲**。不要求字句一致，但禁止把 A 条说成 B 条。

---

## 4. 检查清单（通过 / 不通过）

| 序号 | 检查项 | 说明 |
| --- | --- | --- |
| C1 | 检索相关性 | 至少 1 条 top-k 与问题主题相关；或系统能基于「无/弱检索」给出谨慎结论 |
| C2 | 引用与诚实性 | 答案引用编号与 user 中片段一致，或明确声明当前材料不足以结论 |
| C3 | 与原文不矛盾 | 对照 `data/*.md` 中对应段落，无与原文相反的论断或虚构条号 |

**判定**：C1～C3 均满足则本条问题 **通过**；1～2 个问题均通过则 **MVP 本项验收通过**。
