# v1.1.5 QA 接口限流：默认每分钟 2 条（测试配置）


| 属性   | 说明                                            |
| ---- | --------------------------------------------- |
| 文档版本 | v1.1.5                                        |
| 状态   | 已实施（测试阈值）                                     |
| 适用范围 | `src/qa/webui/app.py` 的 `POST /api/qa/stream` |


---

## 1. 目标

- 为 QA 流式接口增加基础限流，避免单客户端短时间高频请求压垮服务。
- 默认采用测试阈值：**每客户端每分钟 2 次**。
- 限流命中时返回 `HTTP 429` 与明确错误信息。

---

## 2. 配置项

新增配置（`src/conf/settings.py`）：

- `QA_RATE_LIMIT_ENABLED`：是否启用限流，默认 `true`
- `QA_RATE_LIMIT_PER_MINUTE`：每客户端每分钟次数，默认 `2`

`.env.example` 已补充对应注释。

---

## 3. 实现方案

- 限流粒度：`/api/qa/stream` 接口，按客户端 IP（优先 `X-Forwarded-For`，否则 `request.client.host`）。
- 计数窗口：滑动时间窗 60 秒。
- 数据结构：进程内内存字典 `client -> deque[timestamp]`。
- 并发安全：`threading.Lock` 保护读写窗口。
- 超限行为：抛出 `HTTPException(429)`，提示重试秒数。

---

## 4. 已落地代码

- `src/conf/settings.py`
  - 新增 `qa_rate_limit_enabled`、`qa_rate_limit_per_minute`
- `src/qa/webui/app.py`
  - `lifespan` 初始化限流状态容器
  - 新增 `_client_key()`、`_enforce_rate_limit()`
  - 在 `api_qa_stream()` 入口执行限流校验
- `tests/test_qa/test_webui_app.py`
  - 新增限流测试：同客户端连续 3 次请求，第 3 次返回 429

---

## 5. 验收标准

- 默认配置下：同一客户端 1 分钟内前 2 次成功，第 3 次 `429`。
- 关闭开关后：不限流。
- 前端收到 429 时能展示错误，不会卡住（沿用现有错误处理链路）。

---

## 6. 已知边界

- 当前为**单进程内存限流**；多进程/多实例部署下不共享计数。
- 生产环境建议后续升级为 Redis/Nginx/API Gateway 统一限流。
