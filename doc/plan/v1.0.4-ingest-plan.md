## 1. 目标

1. 提供**一条命令**跑通：扫描 [`data/`](../../data/) 下 Markdown → 字符滑窗切分 → BGE-M3 批量编码 → 写入 [`settings.es_index`](../../src/conf/settings.py) 对应 ES 索引。
2. 文档 `_source` 与 [notebooks/c04_es.ipynb](../../notebooks/c04_es.ipynb) 及 [`apply_chunk_source_defaults`](../../src/es_store/chunk_defaults.py) 约定一致，避免笔记本与生产脚本分叉。
3. 验收：脚本成功退出、bulk 无错误、`count` 与 chunk 条数一致；可选 kNN 抽检。

**不在本版本**：增量更新、按删除文件清理 ES 文档、大批量分片 bulk（仅文档说明上限）、问答闭环（见 MVP §9 下一项）。

## 3. 交付形态

### 3.1 CLI

- **路径**：[`scripts/rag_ingest.py`](../../scripts/rag_ingest.py)（脚本内将 `src` 置于 `sys.path`，与 `es_smoke_test.py` 一致）。**不得**使用 `scripts/ingest.py` 作为文件名，否则运行时会与 **`src/ingest`** 包同名冲突。当前 [`pyproject.toml`](../../pyproject.toml) 为 `src` 下多模块、**无** `rag_law` 顶层包，故不强制 `python -m rag_law.ingest`。
- **依赖**：`uv sync --extra embedding`；本机 Elasticsearch 可达（参见 [`doc/storage/docker-compose.elasticsearch.yml`](../storage/docker-compose.elasticsearch.yml)）。

### 3.2 建议参数

| 参数 | 说明 |
| --- | --- |
| `--data-dir` | 默认项目根下 `data` |
| `--recreate` | 默认 **开启**（全量删索引再建）；提供 `--no-recreate` 时仅 `ensure_index(recreate=False)`，文档中注明与旧数据并存风险 |
| `--boundary-aware` | 可选，透传切分（默认 `False`，与多数 notebook 一致） |
| `--dry-run` | 可选：只统计块数/不连 ES、不加载模型 |

## 4. 流水线步骤（实现顺序）

1. `get_settings.cache_clear()` → `get_settings()`。
2. `load_all_chunks(data_dir=..., boundary_aware=...)`；若 0 块则退出码非零。
3. `build_embedder` → `embed_documents([c.text for c in chunks])`；断言 `len(embeddings) == len(chunks)`。
4. `elasticsearch_client` → `ping`。
5. `EsChunkStore(client, settings.es_index, dense_dims=embedder.dense_dimension)` → `ensure_index(recreate=...)`。
6. 每条 chunk 与对应向量组装为 `dict`（字段与 c04 一致），再 `bulk_index_chunks`。
7. `refresh`；`count` 校验等于文档条数。

## 5. `chunk_embedding_to_source`（建议）

在 [`src/es_store/`](../../src/es_store/) 或 [`src/ingest/documents.py`](../../src/ingest/documents.py)（新建）提供纯函数，例如：

`chunk_embedding_to_source(chunk: TextChunk, embedding: list[float]) -> dict[str, Any]`

- 含 `text`、`embedding`、`source_file`、`source_path`、`chunk_index`、`char_start`、`char_end`、`source_doc_id`（来自 `source_id`）、`mime_type`、`doc_type`、`domain`、`chunk_type`、`extra` 等；可再调 `apply_chunk_source_defaults` 或依赖 `bulk_index_chunks` 内统一补齐。
- **笔记本**可择机改为调用该函数，减少重复（可选，不阻塞本计划验收）。

## 6. `source_sha256`（可选首版）

- [v1.0.3-es-store-plan.md](v1.0.3-es-store-plan.md) 已定义 `source_sha256` 字段。
- **推荐实现**：按 **文件** 对 UTF-8 正文字节做 SHA256，同文件各 chunk 写同一值。实现上可用 **按文件遍历**（`glob` → 读入 → `hashlib.sha256` → `iter_file_chunks` → 汇总 chunks），再一次性 `embed_documents`；结构清晰。
- **极简首版**：不计算哈希，依赖默认值 `""`（已由 `apply_chunk_source_defaults` 写入）。

## 12. OSS 来源 Markdown 入库（阿里云）

### 12.1 目标与范围

- 从 **指定 OSS 桶 + 对象前缀** 列举全部 `*.md`，下载到本地固定目录后，执行与 [`scripts/rag_ingest.py`](../../scripts/rag_ingest.py) **相同的** 切分、BGE 向量化、`ensure_index` + `bulk_index_chunks` 流程。
- **替换** 当前 ES 中 chunk 索引数据（默认 `recreate=True`，与本地入库一致）。
- 每条文档 `_source` **补充** `source_oss_url`（虚拟主机样式公网 URL，便于溯源；若桶私有需另行签名/CDN，本段仅记录 URL 构造约定）。
- **不替代** 本章 §1–§11 的本地 `data/*.md` 入库；二者为 **并列入口**，复用 [`src/ingest/`](../../src/ingest/)、[`src/es_store/`](../../src/es_store/)。

### 12.2 入口与依赖

| 项 | 说明 |
| --- | --- |
| 脚本 | [`scripts/ingest_oss_md_to_es.py`](../../scripts/ingest_oss_md_to_es.py) |
| 依赖 | `uv sync --extra oss --extra embedding --extra es`（安装 `oss` extra 时注意不要单独卸掉 embedding/es） |
| OSS 封装 | [`src/utils/oss_aliyun/client.py`](../../src/utils/oss_aliyun/client.py)、[`crud.py`](../../src/utils/oss_aliyun/crud.py)（`list_all_object_keys`、`download_object_to_file`） |
| 公网 URL | [`src/utils/oss_aliyun/public_urls.py`](../../src/utils/oss_aliyun/public_urls.py)（`oss_virtual_host_public_url`） |
| ES 字段 | [`src/ingest/documents.py`](../../src/ingest/documents.py) 的 `chunk_embedding_to_source(..., source_oss_url=...)`；[`src/es_store/mapping.py`](../../src/es_store/mapping.py) 中 `source_oss_url` |

### 12.3 默认约定（CLI / 环境变量可覆盖）

| 参数 / 变量 | 默认 | 含义 |
| --- | --- | --- |
| `--bucket` / `OSS_BUCKET` | `rag-law` | 桶名 |
| `--prefix` / `OSS_OBJECT_PREFIX` | `md3/` | 仅处理此前缀下对象（即桶内 **`md3/` 目录下全部 `.md`**） |
| `--download-dir` | `data/md_minerU` | 下载目录；对象 key 中 `/` 在本地文件名中替换，避免重名覆盖；与 `Settings.oss_download_dir` / `OSS_DOWNLOAD_DIR` 一致 |
| `--limit` | （不传） | 可选；不传则前缀下 **全部** `.md`；传 `N` 则仅按 key 字典序前 `N` 个（调试用） |
| 凭证与地域 | `OSS_REGION`、`OSS_ACCESS_KEY_ID`、`OSS_ACCESS_KEY_SECRET` | 与 OSS SDK V2 推荐命名一致；可选 `OSS_ENDPOINT`（亦写入 `Settings`） |
| 桶与前缀 | `OSS_BUCKET`、`OSS_OBJECT_PREFIX` | 由 [`Settings`](../../src/conf/settings.py) 读取；前缀无末尾 `/` 时自动补全 |

### 12.4 流水线（与 `rag_ingest` 对齐处标注「同」）

```mermaid
flowchart LR
  OSS[ListObjectsV2 前缀下对象 key]
  DL[下载至 data/md_minerU]
  CH[切分 + 每文件 SHA256 同]
  EMB[BGE embed_documents 同]
  ES[ensure_index recreate + bulk 同]
  OSS --> DL --> CH --> EMB --> ES
```

1. **列举**：`list_all_object_keys(bucket, prefix)` → 保留 `*.md` → 按 key 排序 → 可选 `--limit` 截断。  
2. **下载**：对每个 object key 调用 `download_object_to_file`；建立 **`source_path`（相对项目根）→ `source_oss_url`** 映射（由下载路径与 `oss_virtual_host_public_url(bucket, region, key)` 得到）。  
3. **切分**：`load_chunks_with_sha256(None, md_paths=[...])`（**同** `rag_ingest`：`Settings` 控制句边界、语义合并等）。  
4. **向量**：`build_embedder` → `embed_documents`（**同**）。  
5. **写入 ES**：`EsChunkStore.ensure_index(recreate=...)` → `bulk_index_chunks`；组装文档时传入 `source_oss_url`（相对仅本地入库的增量字段）。

### 12.5 验收

- 脚本成功退出、bulk 无错误；`--recreate`（默认开）下 `count` 与本次 chunk 条数一致。  
- 抽检：`kNN` 或 Dev Tools 可见 `_source.source_oss_url`；若需内网/签名访问，在后续版本单独设计，不在本 MVP 范围。
- **不删错、不漏删（共享索引时）**：用 [`ingest.oss_layout.iter_oss_md_ingest_plan`](../../src/ingest/oss_layout.py) 得到本批 `source_path` 列表；[`es_store.scope_replace`](../../src/es_store/scope_replace.py) 的 `preview_chunks_for_source_paths` 入库前审查、`delete_chunks_by_source_paths` 与预览同范围删除后再 bulk。交互式步骤见 [`notebooks/c06_oss_ingest.ipynb`](../../notebooks/c06_oss_ingest.ipynb)。
