# v1.1.0 统一日志（loguru）计划

| 属性 | 说明 |
| --- | --- |
| 文档版本 | v1.1.0 |
| 状态 | 实施基线 |
| 适用范围 | 仓库 `rag_law` 开发与生产排障：统一日志级别、格式、初始化点；与 uvicorn/标准库日志衔接；安全红线与验收 |

---

## 1. 目标与范围

- **目标**：在 MVP 已落地的前提下，引入 **loguru** 作为统一日志门面，通过 **`Settings` 环境变量** 控制级别与输出；在 **QA WebUI** 与 **主要脚本** 入口完成一次性初始化；将 **uvicorn / FastAPI** 使用的标准库 `logging` 输出**拦截**到 loguru，避免两套格式并存。
- **范围**：`pyproject.toml` 主依赖增加 `loguru`；`src/conf/logging_setup.py` 提供 `configure_logging(settings)`；`LOG_LEVEL` / `LOG_FORMAT` / `LOG_FILE` 配置项；WebUI **HTTP 访问**与问答流 **结束摘要**（不含全文用户问题）；**不**在日志中写入密钥、完整 `.env` 或 OSS 密钥。
- **不在本版**：JSONL 专用 sink、Filebeat、Elasticsearch 日志索引、Grafana 面板（见后续 v1.1.1 监控计划）。

---

## 2. 配置键（`conf.settings.Settings`）

| 环境变量 | 含义 | 默认 | 说明 |
| --- | --- | --- | --- |
| `LOG_LEVEL` | 日志级别 | `INFO` | 允许：`DEBUG`、`INFO`、`WARNING`、`ERROR`、`CRITICAL`（大小写不敏感，加载时规范为大写） |
| `LOG_FORMAT` | 控制台/文件行格式 | `text` | `text`：人类可读；`json`：loguru **serialize** 风格 JSON 行（便于后续与 JSONL/采集衔接） |
| `LOG_FILE` | 可选文件路径 | 空 | 未设置或空字符串表示**仅 stderr**；相对路径相对于**项目根**（与 `project_root()` 一致） |

可选后续扩展（本计划不强制实现）：按大小/按天轮转、独立 `LOG_JSON_FILE`（见 v1.1.1）。

---

## 3. 单点初始化：`configure_logging`

- **位置**：`src/conf/logging_setup.py`。
- **行为**：
  - 调用 **`logger.remove()`** 清空 loguru 默认 handler，再按 `Settings` 添加 stderr（及可选文件）sink。
  - **幂等**：进程内重复调用不重复挂载 sink（内部标志位）；测试可提供 `reset_logging_for_tests()` 或 `force=True` 以便重置。
  - **标准库拦截**：安装 **`logging.Handler`** 子类（loguru 文档中的 `InterceptHandler` 思路），将根 logger 及 `uvicorn` / `uvicorn.access` / `uvicorn.error` / `fastapi` 等子 logger 的输出导入 loguru，避免与 uvicorn 自带 handler 格式冲突。

---

## 4. 初始化调用点

| 入口 | 调用时机 |
| --- | --- |
| `src/qa/webui/app.py` 的 `lifespan` | 在 `get_settings.cache_clear()` 之后、`get_settings()` 取得 `settings` 之后**立即**调用 `configure_logging(settings)`，再构建 embedder / ES / OpenAI 客户端 |
| `scripts/rag_ingest.py` | `get_settings()` 之后 |
| `scripts/rag_qa.py` | `get_settings()` 之后 |
| `scripts/llm_smoke_test.py` | `get_settings()` 之后 |
| `scripts/es_smoke_test.py` | `get_settings()` 之后 |

**与 uvicorn 的关系**：以 `uv run uvicorn qa.webui.app:app ...` 启动时，应用导入后由 uvicorn 驱动 lifespan；在 lifespan 内完成 `configure_logging` 即可在**首请求前**完成 loguru 与拦截配置。若将来需要在 import 阶段更早初始化，可将 `configure_logging` 前移到模块顶层（需注意与 `get_settings` 缓存顺序一致）。

---

## 5. 记录内容（最小可行）

- **HTTP**：中间件记录方法、路径、状态码、耗时（毫秒）、`X-Request-ID`（若请求未带则生成并写回响应头）。
- **问答流**：在流结束（`done` 事件）时记录：`retrieval_k`、总耗时、**问题长度**与 **SHA256 前 16 位指纹**（不记录全文 query）。
- **禁止**：`MODEL_API_KEY`、`ES_PASSWORD`、`OSS_ACCESS_KEY_SECRET` 等；仅可记录「已配置 / 缺失」类信息（现有脚本若已脱敏打印 API Key，保持脱敏逻辑，不改为完整密钥）。

---

## 6. 安全与隐私

- 默认不把用户问题全文写入日志；使用长度 + 短指纹满足排障与关联需求。
- 日志文件若配置在磁盘，注意文件权限与备份策略（生产由运维与 v1.1.1 观测栈约束）。
- 测试环境通过 `PYTEST_CURRENT_TEST` 跳过 `.env` 时，日志配置仍只依赖进程环境，与现有 `Settings` 行为一致。

---

## 7. 测试与 CI

- 新增针对 `configure_logging` 的轻量单元测试：`LOG_LEVEL=WARNING`、临时目录文件 sink、或 `reset_logging_for_tests` + `force=True` 避免污染其他用例。
- 现有 WebUI 测试在 lifespan 内会触发一次 `configure_logging`；保持 `LOG_LEVEL` 默认或设为 `WARNING` 可减少刷屏（由 monkeypatch 注入环境变量）。

## 8. 验收清单

- [ ] `uv run uvicorn qa.webui.app:app --host 127.0.0.1 --port 8766` 启动后，控制台日志格式统一，调整 `LOG_LEVEL` 可见级别变化。
- [ ] 设置 `LOG_FILE` 后，同内容写入指定文件（路径相对项目根解析正确）。
- [ ] `uv run python scripts/rag_ingest.py --dry-run` 与 `uv run python scripts/rag_qa.py "测试" --dry-run` 无异常，无密钥泄露。
- [ ] `pytest` 全绿，且日志相关用例不依赖本机真实 `.env` 密钥。

---

## 9. 与后续 v1.1.1 的衔接

v1.1.0 固定 **`configure_logging` 契约**与字段名后，v1.1.1 可在同一函数中**增加第二个 sink**（仅 JSONL 文件）、`enqueue=True`、字段契约与 Filebeat 示例，而不必再改各入口调用顺序。
