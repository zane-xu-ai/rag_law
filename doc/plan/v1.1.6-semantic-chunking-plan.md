- 通过相邻块语义相似度判断，把过短且语义连续的块合并，减少语义断裂。
- 保持默认行为不变（开关默认关闭），支持灰度启用。

## 2. 实现概览

- 新增配置（`Settings`）：
  - `CHUNK_SEMANTIC_MERGE_ENABLED`（默认 `false`）
  - `CHUNK_SEMANTIC_MERGE_THRESHOLD`（默认 `0.82`）
  - `CHUNK_SEMANTIC_MERGE_MIN_CHARS`（默认 `220`）
  - `CHUNK_SEMANTIC_MERGE_MAX_CHARS`（默认 `2200`）
- 新增算法：
  - `semantic_merge_chunks(chunks, ...)`：仅合并**相邻块**
  - 相似度：字符 2-gram 余弦相似度（轻量、无额外模型依赖）
  - 合并条件：当前块较短（低于 `MIN_CHARS`）且合并后不超过 `MAX_CHARS`，并且相似度达到阈值
- 接入链路：
  - `iter_chunks_for_text` / `iter_file_chunks` / `iter_chunks_for_data_dir`

  - 记录 `chunk_semantic_merge_done`，含 `before_count` / `after_count` / `merge_ratio` / `threshold_hit_rate`

## 2.1 字符 n-gram + 余弦相似度原理（详细）

当前实现使用**字符 2-gram**（即 n=2）做轻量文本表示，再用余弦相似度判断两块是否“语义接近”。

### 步骤A：把文本拆成 2-gram 片段

以文本 `合同纠纷的诉讼时效` 为例，2-gram 序列为：

- `合同`、`同纠`、`纠纷`、`纷的`、`的诉`、`诉讼`、`讼时`、`时效`

每个 2-gram 统计出现次数，形成一个稀疏“词袋向量”（`Counter`）。

### 步骤B：计算两个块的余弦相似度

设两块向量分别为 `A`、`B`，相似度公式：

- `cos(A,B) = (A·B) / (||A|| * ||B||)`

解释：

- 分子 `A·B`：共享 2-gram 越多，值越大；
- 分母是各自长度归一化，避免“文本更长就天然更像”的偏差；
- 结果范围约在 `[0,1]`（本场景计数非负）。

### 步骤C：是否合并的判定

并非“相似就合并”，还需同时满足长度约束：

- 当前块长度 `< CHUNK_SEMANTIC_MERGE_MIN_CHARS`
- 合并后长度 `<= CHUNK_SEMANTIC_MERGE_MAX_CHARS`
- 相似度 `>= CHUNK_SEMANTIC_MERGE_THRESHOLD`

仅在以上条件都满足时，才将**相邻块**合并。

## 2.2 举例：句边界切分 vs 语义合并后

假设输入文本：

1) `合同纠纷中，诉讼时效通常为三年。`
2) `合同纠纷的时效中断后，期间重新起算。`
3) `劳动争议仲裁时效通常是一年。`

并假设句边界切分后得到 3 个相邻块（示意）：

- 块A：合同纠纷诉讼时效（三年）
- 块B：合同纠纷时效中断与重新起算
- 块C：劳动争议仲裁时效（一年）

### 仅句边界切分（旧行为）

- 保持 A / B / C 三块不变。

### 启用语义合并（当前行为）

- A 与 B：共享大量 2-gram（如“合同”“纠纷”“时效”），相似度高；
- B 与 C：主题切换明显，共享 2-gram 少，相似度低。

因此结果通常为：

- A+B 合并为一个更完整“合同时效”语义块；
- C 保持独立。

这会减少语义断裂，提升检索命中后提供给 LLM 的上下文完整性。
